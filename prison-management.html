Prison Management System

Academic Project | Manila, 2024


View on GitHub


Project Overview


A database design project focused on modeling a complete Prison Management System using enhanced entity–relationship diagrams (EERD) and normalized relational structures. The system captures inmate records, personnel information, facility operations, and role-based user access.



Tech Stack


  • Tools: MySQL Workbench, Draw.io, ER Diagramming Tools
  • Languages: SQL
  • Concepts: Normalization (1NF–3NF), Referential Integrity, Primary/Foreign Keys


Key Features


  • Designed an enhanced entity–relationship diagram (EERD) modeling inmates, officers, facilities, cases, and cell assignments.
  • Normalized all relations to reduce redundancy and ensure efficient queries.
  • Implemented serialization utilities and core classes (e.g., User) for storing and retrieving role-based data.
  • Enforced referential integrity using composite keys, constraints, and optimized table relationships.
  • Created a foundation suitable for future system expansion such as visitation logs, schedule tracking, and inmate classification.


Results


  • Produced a clean, normalized, and scalable database structure suitable for a real operational prison information system.
  • Reduced potential data redundancy by applying proper normalization up to 3NF.
  • Ensured accurate data linking between inmates, officers, cases, and facilities through well-defined keys and constraints.
  • Provided a solid blueprint for implementing the full backend system in future development.
